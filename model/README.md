# LLLM Meta-Model

This folder defines the canonical information model for the LLLM enterprise architecture and AI infrastructure programme.

The meta-model is the source of truth from which procurement workbooks, Coda tables, Logseq graphs, GitHub documentation, configuration registers, asset registers, warranty registers, knowledge pages, diagrams, and operations manuals should be generated.

## Contents

- `LLLM_Meta_Model_Specification_v0.1.md` — governing specification.
- `entity-catalogue.md` — canonical entity definitions.
- `relationship-catalogue.md` — relationship vocabulary.
- `attribute-catalogue.md` — standard attributes.
- `identifier-standard.md` — universal ID system.
- `lifecycle-standard.md` — lifecycle states and transitions.
- `generation-standards.md` — rules for generating views and documents.

## Principle

Do not manually duplicate facts across documents. Model the fact once, then generate the appropriate view.
